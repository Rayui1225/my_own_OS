Milestone 5A

Physical Memory Manager Flow

This diagram explains the first physical frame allocator. The implementation starts with a hardcoded usable RAM range, subtracts the kernel image range from linker symbols, splits the remaining memory into 4 KiB frames, and stores those frames in a stack allocator.

Memory Source

map.rs owns where memory information comes from. Today it is hardcoded; later it can be backed by device tree parsing.

Frame Unit

Frame represents a page-aligned 4 KiB physical frame, not just any raw address.

Strategy Boundary

FrameAllocator is the small trait boundary. The first implementation is StackFrameAllocator.

Range Subtraction

The usable RAM range includes the kernel image at the beginning. PMM initialization removes the reserved kernel region before creating allocatable frames.

1

Read the Current Memory Map

pub fn usable_memory_ranges() -> &'static [MemoryRange]

The allocator does not hardcode QEMU memory constants by itself. It asks the memory map layer for usable ranges.

Design point: the hardcoded provider is isolated, so a future device-tree provider can replace it with less churn.
2

Reserve the Kernel Image

PROVIDE(skernel = .); ... PROVIDE(ekernel = .);

Linker symbols describe the actual kernel image range. Rust reads those symbols and aligns the end upward to the next page boundary.

Why it matters: as the kernel grows, the allocator automatically reserves the right amount of memory.
3

Subtract Reserved Memory

for segment in subtract_reserved(range, reserved) { if !segment.is_empty() { PMM.add_range(segment); } }

A single usable range minus a single reserved range can produce at most two remaining segments: one before the reserved area and one after it.

Shape: [usable] - [reserved] becomes [left free] and [right free].
4

Split Segments into Frames

let aligned = range.align_inward(); while current < aligned.end { self.push(Frame::from_start_address(current)); current += PAGE_SIZE; }

PMM only manages full frames. Each remaining segment is aligned inward, then sliced into 4 KiB frame starts.

Frame rule: partial pages are ignored, because allocation happens at page granularity.
5

Use Stack Allocation

alloc_frame() -> pop from stack dealloc_frame(frame) -> push back

The first allocator is intentionally small and direct. It stores free frame addresses in a static array and uses len as the stack top.

Current behavior: because frames are pushed from low to high addresses, the first allocation returns a high address.

What Can Change Later

  • map.rs can move from hardcoded ranges to device tree parsing.
  • StackFrameAllocator can be replaced or compared with a bitmap allocator.
  • The public PMM API can stay stable while the allocation strategy changes underneath.
  • A future free_frame_count() can report current availability separately from total discovered frames.
