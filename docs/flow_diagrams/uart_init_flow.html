Milestone 1

UART Initialization Flow

This diagram explains what our current uart::init() actually does on a 16550-compatible UART. The goal is not to memorize magic numbers, but to understand which hardware state each step is trying to establish.

Big Picture

We are preparing the serial device so later println! can turn Rust text into bytes on the QEMU serial console.

Why It Exists

Without this setup, UART may still be in an unknown state, so writing bytes might not show anything visible.

Mindset

Think of this as a hardware bring-up checklist: disable surprises, choose a line format, clear buffers, then send text.

1

Disable Interrupts

write_reg(IER, 0x00);

We turn off UART-generated interrupts first, because this milestone uses simple polling-style output rather than interrupt-driven I/O.

IER means Interrupt Enable Register. It controls which UART events are allowed to interrupt the CPU.
2

Enter Baud-Rate Setup Mode

write_reg(LCR, LCR_DLAB);

We temporarily switch the UART into a mode where register offsets 0 and 1 no longer mean normal transmit/interrupt registers. They become the divisor-latch registers instead.

LCR means Line Control Register. DLAB means Divisor Latch Access Bit.
3

Program the Baud-Rate Divisor

write_reg(DLL, 0x01); write_reg(DLM, 0x00);

These two writes store the baud-rate divisor as a 16-bit value. In our current code, the divisor is 1, which is enough for this QEMU-based learning setup.

DLL and DLM mean Divisor Latch Low and Divisor Latch High.
4

Return to Normal Line Format

write_reg(LCR, LCR_8N1);

After programming the divisor, we switch back into normal line-control mode and choose the classic serial format: 8 data bits, no parity, 1 stop bit.

8N1 is serial shorthand for 8 data bits, No parity, 1 stop bit.
5

Enable and Clear FIFOs

write_reg(FCR, 0x07);

This turns on the internal FIFO queues and clears any stale data that may already be sitting inside the UART buffers.

FIFO means First In, First Out. It is the UART's small internal queue for incoming and outgoing bytes.
6

Mark the Port as Ready

write_reg(MCR, 0x03);

We set traditional modem-control readiness bits. In QEMU this is mostly part of the common 16550 bring-up recipe, but on real serial hardware these bits can matter more.

MCR means Modem Control Register. The usual ready-style bits here are DTR and RTS.

Reading Notes

  • This sequence is not a universal law for every UART. It is a small, practical recipe for a 16550-compatible device.
  • The values themselves are less important than the intent behind each step: disable interrupts, configure timing, configure frame format, clear buffers, mark ready.
  • For Milestone 1, this is enough. We are not yet doing input handling, interrupts, or advanced flow control.
  • The real teaching value is seeing that driver initialization usually means “write a specific state into several control registers.”

Name Cheat Sheet

THR

Transmit Holding Register. Writing one byte here asks the UART to send it.

LCR

Line Control Register. Controls how each serial frame is shaped.

DLL / DLM

The low and high halves of the baud-rate divisor. Together they decide serial timing.

FCR

FIFO Control Register. Used to enable and clean internal buffering.

MCR

Modem Control Register. Extra readiness and control bits for serial-port behavior.
